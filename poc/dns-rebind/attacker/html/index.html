DNS-rebind PoC — attacker page

This page is served from . When you visit it, the browser caches the DNS resolution of evil.example as the attacker's IP. After the 1-second TTL, the browser re-resolves and now gets the victim's IP — but the page's origin is still evil.example:3000, so the fetch is considered same-origin under SOP.

If mcp-streamablehttp-proxy v0.2.0 is bound to localhost:3000 with no Origin / Host validation (which it is by default), the post-rebind fetch will succeed.
